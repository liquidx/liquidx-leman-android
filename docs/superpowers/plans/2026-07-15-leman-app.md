# LeMan App Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> The normative spec lives in `docs/spec/01…08` and the design handoff in `docs/design/README.md`. Every task below cites the spec sections it implements; when this plan and the spec disagree, the spec wins.

**Goal:** Implement the complete LeMan Android app — a mobile client for the Hermes agent gateway — per `docs/spec/`: data layer (Room system-of-record, Hermes run client, SSE fold), custom Compose design system, four screens, markdown renderer, debug tooling, and test suites.

**Architecture:** Single `:app` module, package-by-layer under `net.liquidx.leman` (domain / data / ui / debug). Manual DI via `AppContainer`. Unidirectional flow: Room `Flow` + ConnState + prefs + streaming run state → `StateFlow<UiState>` → Compose. The only live socket is a run's event stream.

**Tech Stack:** Kotlin 2.2, Compose BOM 2025.12, AGP 9.2.1, OkHttp, kotlinx-serialization, Room+KSP, DataStore, navigation-compose, commonmark-java, JUnit4/Turbine/MockWebServer/Robolectric/Roborazzi.

## Global Constraints

- Package `net.liquidx.leman`; minSdk 34, targetSdk/compileSdk 36.
- Base URL default `https://api.gent.ino.ink`; auth `Authorization: Bearer <key>`; model id sent is always `hermes-agent`.
- Gateway has **no thread store**: no `GET /v1/runs` list, no `/v1/threads`, no global SSE. History is client-sent on each run as `input=[{role,content}]`. `session_id` is correlation only.
- SSE framing is **`data:`-only** (no `id:`/`event:` lines; type is `.event` inside JSON; ends `: stream closed`). Reconnect replays from start → always reset accumulated text before replay.
- Colors/type/motion exactly per `docs/spec/06` `LemanColors`/`LemanType`/`LemanMotion` tables. Font Spline Sans Mono 400/500/600 bundled in `res/font/`. No rounded corners, no shadows except `Modifier.glow`. No Material components in the visual tree.
- Errors: sealed `ApiError`; user actions never auto-retry; backoff 1s→30s cap ±20% jitter for probes/stream reconnect only. No toasts/snackbars/dialogs.
- `Json { ignoreUnknownKeys = true }`; unknown statuses/events → `Unknown`, never crash.
- Debug-only code in `app/src/debug/` or behind `BuildConfig.DEBUG`; `applicationIdSuffix = ".debug"`; API key masked in all logs.
- Tests: `methodOrBehavior_condition_expectation`; virtual time, no sleeps; fixtures under `app/src/test/resources/fixtures/` shared by unit + screenshot + fake seed.
